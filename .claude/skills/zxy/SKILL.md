# 公众号自适应排版 Skill

把文章排成可粘贴进微信公众号、**粘贴后样式不丢**的 HTML。

**不用固定主题库，每篇由你根据内容自主设计风格。** 本 skill 完全自包含——风格素材在自带 references、校验脚本自带，可独立分发、单独运行，不依赖任何其他 skill。

## 核心原则

1. **微信兼容是硬约束，自主抉择是自由度**。配色/版式/组件随便选，但 HTML 必须满足微信编辑器兼容规则（见下"红线" + 详读 [references/wechat-compat.md](references/wechat-compat.md)）。不兼容的 HTML 粘进去样式全丢——这是底线，不可商量。
2. **风格服务于内容**。先读懂题材、语气、情绪、读者，再决定气质（克制 or 浓烈、冷 or 暖、严肃 or 活泼）。不要所有文章一个样。
3. **每篇解释你的选择**。交付时说明"为什么这个题材用这套配色/版式"，让用户可审计、可调整。

## 工作流

### 1. 读懂内容，定调

通读全文，判定并写进决策说明：
- **题材**：教程 / 测评 / 观点 / 故事 / 盘点 / 复盘 / 科普 ……
- **语气**：理性/感性、克制/张扬、轻松/严肃
- **情绪温度**：冷（科技/数据）/ 暖（生活/情感）/ 中（知识科普）
- **读者** 与 **关键视觉锚点**（核心金句、数据、对比、步骤）

### 2. 抉择配色与版式

据上一步定调，从 [references/style-library.md](references/style-library.md) 取灵感或自创一套：
- **配色**：1 主色 + 1 强调色 + 中性背景/文字色（3–5 色足矣）。主色呼应内容气质（中医→本草赭红/茶褐；科技→冷青/墨蓝；口水战→朱红+警示橙；随笔→黑白灰）。
- **版式密度**：信息密度高→紧凑组件+清单；叙事→大留白+长行距。
- **强调手段**：定 1 种主强调（左竖条标题 / 色块标签 / 下划线关键词）+ 1 种次强调，全文统一，别堆。

### 3. 装配 HTML（遵守微信兼容）

用 style-library.md 的组件库（或自创等价组件）装配。**每个组件必须**：
- 样式全部内联（`style="..."`）
- 所有中文文字节点用 `<span leaf="">文字</span>` 包裹
- 容器用 `<section>`，不用 `<div>`
- 中文标点全角（，。！？""''（）—— ……）；代码块内除外

### 4. 校验（强制，不可跳过）

写完文件后跑校验脚本，ERROR 必须清零：

```bash
<SKILL_ROOT>/scripts/validate_html.py <生成的.html>
```

- ERROR = 粘贴后会被过滤或样式失效，**必须修**。
- WARNING（半角标点、漏 `<span leaf>`）也修到 0 再交付——漏 leaf 是最高频返工点。

### 5. 交付（默认两个文件）

1. **干净正文** `{名}_公众号.html`：纯 `<section>…</section>` 片段，**不包 `<!DOCTYPE>/<html>/<head>/<body>`**。**校验脚本只跑这个文件**，必须 ERROR 清零。
2. **带复制按钮的预览页** `{名}_预览.html`：把干净正文嵌进 [references/preview-template.md](references/preview-template.md) 的外壳，浏览器打开后点右上角「复制到公众号」一键复制 section 富文本。**不跑校验**（外壳含 div/script/id，是浏览器用的，不进公众号）。

告知用户：**打开 `{名}_预览.html` → 点「复制到公众号」→ 公众号编辑器 Ctrl/⌘+V 粘贴**；干净正文文件作为兜底。附**校验结论 + 风格决策说明**（配色/版式/为什么）。

## 微信兼容红线（核心，详读 references/wechat-compat.md）

- **禁止**：`<style>`/`<script>`/`<div>`/`<link>`、`class`/`id`、`position:fixed/absolute/sticky`、`float`、`@media`/`@keyframes`/`@import`、`display:grid`、CSS 变量 `var(--x)`、外部字体、伪元素 `::before/::after`。
- **必须**：样式全内联 `style`；**所有文字节点 `<span leaf="">` 包裹**（否则粘贴后样式丢失，最常见致命错）；容器用 `<section>`；中文全角标点。
- **可用**：`display:flex`（慎做多栏，手机端挤）、`linear-gradient`、`border-radius`、`box-shadow`、`<section>/<p>/<span>/<strong>/<img>/<h3>`。

## 自主抉择决策表（起点，可突破）

| 内容气质 | 配色方向 | 版式 |
|---|---|---|
| 科技/数据/客观 | 冷色：墨蓝/青绿/石墨灰 | 紧凑、左竖条标题、数据块 |
| 知识科普/教程 | 中性+一抹主色：靛蓝/森绿 | 步骤化、代码块、清单 |
| 故事/情感/生活 | 暖色：米白/赭红/茶褐/暖橙 | 大留白、长行距、引文卡 |
| 观点/评论/争议 | 强对比：朱红/深墨+警示橙 | 对阵块、优缺点堆叠、金句高亮 |
| 极简/随笔 | 黑白灰+单色点缀 | 大量留白、细分隔线 |

**匹配内容 > 遵守表格**。中医科普用本草风、口水战用对阵块——这是本 skill 区别于固定模板的价值。

## 输出格式硬要求

- **粘贴用内容**是 `<section>…</section>` 正文片段，**不带文档外壳**。校验只跑这个片段。
- 完整可独立打开的 HTML（带 `<!DOCTYPE>`，即预览页）仅供浏览器预览 + 一键复制；公众号只收其中 section。
- 文件名：干净正文 `{名}_公众号.html`，预览页 `{名}_预览.html`。

## Gotchas（真实踩过的坑）

- **漏 `<span leaf>` 是最常见致命错**——粘贴后样式整片丢。靠校验脚本兜底，别跳过。每段、每个标题、每个标签里的中文都要包。
- **别用 flex 做多栏并排**：公众号手机端窄，多栏会挤塌。优缺点/对比改上下堆叠。
- **一篇文章一套配色**：别为了"丰富"塞三种主色，等于没有重点。主色贯穿，强调色只用 2–3 处。
- **强调手段统一**：全文用同 1–2 种（如左竖条标题 + 主色加粗），不要这段左竖条、那段虚线框、又那段渐变高亮。
- **图片**：`<img style="max-width:100%;height:auto;display:block;margin:0 auto">`，不用 `width:100%`（小图会糊）。
- **代码块**：每行一个 `<p style="margin:0">`，不用 `white-space:pre`（会渲染出大空白），缩进用全角空格 `　`。
- **不要文档外壳**：`<!DOCTYPE>/<html>/<head>/<body>` 会被公众号丢弃或干扰粘贴，产物就是裸 section。
- **字体别指望**：公众号基本忽略 `font-family`，写了无害但别依赖特定字体效果。
