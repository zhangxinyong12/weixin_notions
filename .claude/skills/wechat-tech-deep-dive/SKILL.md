# 技术概念讲透：写作规则

## 定位

AI 全栈技术分享。目标读者是用 AI 做开发的程序员，想真正理解技术原理，不想看科普水文。

标准：读者看完一篇文章，从"听说过这个概念"变成"真正理解了它是怎么回事"。

## 核心写作原则

### 1. 逐步推导，不跳步

每步建立在前一步之上。结构用"第一步/第二步/第三步"或自然的问题递进，不用"首先/其次/最后"。

每节末尾制造一个自然的疑问，引出下一节。不是"下面我们来介绍X"，而是前一步的解释本身会产生一个问题。

### 2. 具体先于抽象

先展示一个读者能看见、能运行、能触摸的东西（API 返回值、代码片段、控制台输出），再解释它意味着什么。

不要先给定义再举例子。先给例子，让读者自己感觉到规律，再给名称。

### 3. 每个断言有具体数字

不说"效果很差"，说"相似度只有 0.96"。
不说"差距很大"，说"存储减少 12 倍，质量只掉了不到 1%"。
不说"广泛应用"，说"每周 npm 下载量超过 120 万"。

数字让文章有锚点，也让读者能自行验证。

### 4. 破除至少一个流行误解

每篇文章至少打破一个读者可能相信但实际不对的说法。用真实数据说话：

- "king - man + woman = queen" → 实测只有 0.568 相似度，换类比就崩了
- "更多维度更好" → 256 维的 large 比老模型 1536 维还好
- "Embedding 理解语义" → "好"和"不好"相似度 0.88

### 5. 坦诚局限性

专门用一节写"它做不到什么"。这是深度文章跟科普水文的分水岭。

局限性要用具体的失败案例展示，不是笼统地说"还有改进空间"。

### 6. 不做个人叙事包装

不写"我踩了个坑"、"我花了三天才搞明白"。不是不能提个人经历，但文章的主体是技术本身，不是"我"。

技术解释自己就是内容，不需要"我的故事"来撑。

### 7. 代码要能跑

所有代码片段必须是可运行的，不是伪代码。包括 import 语句、必要的变量声明。

如果代码需要安装依赖，在代码前说明。

### 8. 公式要用"人话"解释

展示数学公式，但紧接着用一句话翻译成人类语言。

```
L = -log(exp(sim(a, p) / τ) / Σ exp(sim(a, n_i) / τ))
```

用人话说：给定一段文本，和它相似的一段文本，以及一堆不相关的文本，要求模型把相似的拉高，不相关的压低。

## 文章结构模板

```
# [概念] 到底在做什么

[开头：直接问读者是否用过这个东西，然后提出本文要回答的核心问题]

## 第一步：[展示最具体的入口]
调用 API/写代码，你拿到了什么。展示原始输出。

## 第二步：[解释输出的含义]
这些数字/结果意味着什么。建立直觉。

## 第三步：[破除一个误解]
用数据测试一个流行说法。

## 第四步：[解释底层原理]
为什么会这样。训练过程/设计决策/数学原理。

## 第五步：[局限性]
它做不到什么。具体的失败案例。

## 第六步：[实用指南]
模型/方案/参数怎么选。有立场的推荐，不是中立罗列。

## 总结
回到开头的核心问题，用一两句话说清答案。
```

这是默认结构，不是固定模板。根据主题可以调整步数和顺序。

## 写作风格

- 短段落。1-3 句一段。
- 段落之间用空行分隔。
- 小标题用问句或动作（"怎么算距离近"、"为什么语义相近的东西会离得近"），不用名词堆砌（"距离度量方法"、"对比学习原理"）。
- 加粗只用于首次定义术语或真正需要强调的结论，不超过全文 5 处。
- 表格只在承载了真正的数据对比时使用，不用表格来罗列概念。
- 代码块只放关键片段，不放完整项目代码。

## 禁止

- "在当今快速发展的时代"、"随着XX的发展"
- "值得注意的是"、"需要指出的是"
- "不仅是X，更是Y" 否定式排比
- 每段结尾都是金句
- 中立罗列选项不给判断
- 空洞的"希望对你有帮助"式结尾
- 三段式并列（连续三个结构相同的句子或段落）
- 强行幽默或自嘲（技术文章不需要硬塞幽默）

## 研究要求

写之前必须做深度调研。验证关键数据点的准确性。

需要验证的内容：
- API 的确切参数和返回值（查官方文档）
- 性能数据的具体数字（查基准测试）
- "大家都知道"的说法是否真实（找反例或原始论文）

调研来源优先级：官方文档 > 基准测试 > 技术博客 > 论文 > 教程。

## 质量检查

写完后自检：

1. 读者看完能不能向别人解释清楚这个概念？
2. 每个断言有没有具体数字或代码支撑？
3. 有没有打破至少一个流行误解？
4. 局限性部分有没有具体的失败案例？
5. 有没有连续三个结构相同的段落？
6. 结尾是不是在回答开头提出的问题？
