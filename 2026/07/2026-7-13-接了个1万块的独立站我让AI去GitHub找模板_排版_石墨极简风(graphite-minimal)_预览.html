QUOTE

接单早就不是“写代码”了，是 对齐需求、找轮子、在轮子上改 。

—— daydayago

前两天接了个独立站的小单子，1 万块。

客户需求一句话就说完了：想做个独立站。

放半年前，我第一反应是开始算账。

要几个页面？要不要后台？数据存哪？域名部署谁管？算下来，1 万块这个价，可能不太够，得砍需求或者加钱。

但这次我没这么想。

我把整个活拆成了三步，每一步都让 AI 干主体。

预计一周搞定，实际可能更快。

本文看点

01

把公司材料丢给 AI，让它自己长出官网

02

去 GitHub 找现成模板，简单二开

03

守住边界，1 万块才赚得到

01

BRIEF

别让客户描述，把他公司的东西丢给 AI

这一步我之前讲过用 AI 画原型对齐需求。这次往前再走一步，玩得更狠。

以前是：客户口头描述需求，我根据描述让 AI 画原型。

问题是客户描述自己的产品，未必比他公司的宣传材料更准。“做得高端一点”“科技感强一点”——这种词，每个人脑子里想的都不一样。

更狠的办法：别让客户描述了。

让他把公司现有的东西直接发过来——产品说明书、宣传册、PPT、现有的老官网、产品图册，哪怕几张参考图，什么都行。

把这些一股脑丢给 AI，让它去读、去理解。

AI 读完，自己就能抓出产品的调性：这是个高端品牌还是性价比工具？面向 B 端还是 C 端？风格该冷峻一点还是温暖一点？

然后让它根据这个理解，直接出官网的原型和样式。

好处很直接：原型不是你拍脑袋画的，是从客户真实的产品里“长”出来的。客户一看，“对，这就是我们公司的感觉”。

这一步省下的，不只是返工，还有你跟客户互相猜心思的时间。

基础的原型对齐，看我之前那篇《接了 5 单才懂单干和上班的区别》。这次是升级版。

02

SEARCH

让 AI 去 GitHub 找现成的轮子

原型和样式都定了，下一步不是打开编辑器写代码。

是找。

我把原型图丢给 AI。上一步它已经吃透了产品的调性，我直接让它按这个风格，去 GitHub 上找结构最接近、气质也搭的开源模板。

这一步以前我从没想过。

以前接单，习惯性思维就是从零开始。建项目、配依赖、搭路由、写组件，光搭架子就要一两天。

但其实你想要的东西，GitHub 上早有人做过了。

独立站这种需求尤其明显。官网模板、落地页模板、电商模板、个人主页模板，NextJS 生态里一抓一大把，很多还维护得不错。

让 AI 在几百个模板里帮你挑出最接近原型的那个，比你自己一个个翻 README 快太多了。

「90% 的独立站需求，GitHub 上都有现成的轮子。没必要自己造。」

03

CUSTOMIZE

在模板上简单二开

找到模板，clone 下来。

剩下的活，就是按原型的样子去改。

换文案、调配色、换图片、加几个页面、改几个组件。

听着简单，其实有个隐藏的门槛：你得先读懂别人怎么写的。

模板不是你的代码。它的目录结构、它用的状态管理、它封装组件的方式，都是别人的习惯。上手就改，很容易改崩。

这也是我以前不爱用模板的原因——读别人的代码，比自己写还累。

但现在 AI 能帮你读。

把整个项目丢进去，问它：这个文件干嘛的、数据怎么流动、我想加个 XX 功能该改哪里。它能给你指路。

“二开”的门槛，被 AI 拉低了一大截。

以前是“读懂代码 + 改代码”，两件事都费劲。

现在是“AI 帮你读懂 + 改代码”，主要费劲的就剩改了。

04

REALITY

但第一版交了，麻烦才刚开始

上面这套流程，做出第一版很快。

但你要清楚：第一版交出去，才是真正费劲的开始。

客户看到东西“诶，像模像样”，立马就开始提意见：这个按钮换个颜色、这个间距太大、这个版式再调调、这里能不能加个动画。

每一句都不大。但架不住多。一轮两轮改下来，三五天就没了。

前面 AI 帮你省下的时间，全搭在这些 UI 微调上。

所以我说 1 万块这个价，勉勉强强。

前提是——你得把客户管住。

接单的时候，话就得讲死：交付包含几轮修改、超出怎么算、UI 大改算新需求、加页面另收费。

这不是抠门。

是 AI 让你出活快，但不等于你的时间不值钱。

「利润不是光靠省出来的，是靠守住边界守出来的。」

05

ESSENCE

接单这件事，本质变了

把三步串起来看，我发现接单的逻辑变了。

以前接单，是写代码。按工时算钱。

现在接单，是拼装。对齐需求 + 找轮子 + 在轮子上改。

1 万块的独立站，纯手撸，我可能要两周，累，还不怎么赚钱。

用这套流程，一周甚至三五天。利润率完全不是一个量级。

而且这不是个例。

小程序、落地页、内部工具、个人主页，这些“看起来要写一堆代码”的活，本质都能套这个流程：先用 AI 把需求具象化，再去找现成的项目，最后做最小改动。

「真正难的从来不是写代码，是把客户脑子里那个模糊的东西，变成一个具体的、能交付的东西。」

AI 让这件事变得很便宜。

06

VALUE

程序员的价值在挪位置

聊到这，我想说个更大的判断。

当 AI 能帮你画原型、找模板、读懂别人的代码、改样式，一个程序员的价值到底在哪？

我觉得在挪位置。

从“我会写”，挪到“我知道找什么、拼什么、哪样改是对的”。

写代码这个动作，门槛在被拉平。

但“判断一个需求该用什么轮子”、“评估一个模板能不能撑住客户后面的需求”、“知道哪里该自己写、哪里该用现成的”——这些判断，AI 暂时替不了。

这套流程里，真正值钱的是我做的两个决定。

一个是“不从零写，去找模板”。

一个是“找来的模板，怎么改才不会埋雷”。

写代码被 AI 接走了，这两个决定，还得我自己拍。

∞

THE END

最后

这套流程我打算固定下来。

往后接这种小单子，第一反应不再是从零评估工期，而是先问一句：这玩意儿 GitHub 上有没有现成的。

有，就接，走三步。

没有，再考虑值不值得自己写。

下一单，我想试试把报价再提一档。

反正交付时间已经被压下来了，省下来的时间，就是利润。


END
