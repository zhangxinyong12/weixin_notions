# 我用 AI 写公众号的完整流程，每天只要几分钟

> 写公众号这件事，我曾经一周都憋不出一篇。现在每天几分钟就能产出一篇。区别不是变勤奋了，是流程变了。

我写公众号有一阵子了。

最开始最痛苦的，不是没东西写。

是写完之后那一堆零碎。

选题、写初稿、配图、排版、调样式、发布。

每一件都不难，但串起来特别消耗精力。

经常是写完一篇，下次再想写就犯懒。

后来我用 AI 把这个流程重新走了一遍。

发现真正费时间的，其实只有两件事。

一是想清楚写什么。

二是把内容变成公众号能看的排版。

这两件事之外的动作，几乎都可以交给 AI。

现在我每天大概花几分钟，就能产出一篇能发的文章。

今天把整套流程拆给你看。

## 整个流程长什么样

先看全貌。

一次性的准备：

```text
装 Node.js
装 Claude Code
配置智谱 GLM
装好排版 skill
```

每天要做的事：

```text
选一个主题
让 AI 生成文章
打开生成的 HTML
复制到公众号发布
```

准备阶段只做一次。

做完之后，每天就是后面那四步。

下面分别说。

## 第一步：装 Node.js

Claude Code 是个命令行工具，跑在 Node 上。

所以先得有 Node。

去官网下一个 LTS 版本，一路下一步装完就行。

装完在终端里敲一下：

```bash
node --version
```

能打印出版本号，就说明装好了。

这一步没什么坑，跳过。

## 第二步：装 Claude Code

Claude Code 是 Anthropic 官方的命令行 AI 助手。

它不是网页版那种聊天框。

它是直接在你电脑的终端里跑，能读你的文件，能调用各种技能。

安装就一行命令：

```bash
npm install -g @anthropic-ai/claude-code
```

装完验证一下：

```bash
claude --version
```

到这里，工具本身就装好了。

但还跑不起来。

因为它需要一个模型在背后回答问题。

这就来到下一步。

## 第三步：配置智谱 GLM

Claude Code 默认用的是 Anthropic 自己的模型。

但我在国内，直接用有两个问题。

一是网络不通，二是按量计费不便宜。

所以我换成智谱的 GLM。

智谱提供了一个和 Claude Code 兼容的接口。

也就是说，**Claude Code 这个壳子不变，背后回答问题的模型换成国产的 GLM。**

好处很直接。

国内直连，不用折腾网络。

价格便宜很多。

中文效果也够用。

配置方法是设两个环境变量：

```bash
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/anthropic"
export ANTHROPIC_AUTH_TOKEN="你的智谱 API Key"
```

API Key 在智谱开放平台注册后就能拿到。

设完之后，在终端里敲 `claude`，能正常对话，就说明通了。

到这一步，AI 这部分就齐了。

能聊天，能读文件，能写文章。

但还差最后一块。

## 第四步：用排版 skill 把文章变成 HTML

这是整个流程里我最想说的部分。

AI 帮我把文章写出来，内容是 Markdown。

但微信公众号不认 Markdown。

直接粘进去，标题不是标题，重点不突出，代码块没法看。

这个坑我之前专门写过。

所以中间必须有一步：**把 Markdown 转成微信能识别的富文本 HTML。**

这一步我现在不让 AI 现场发挥，而是用一个固定的 skill。

这个 skill 叫 gzh-design。

它做的事很固定。

读进一篇 Markdown。

按一套预设的样式，转成可以直接粘进公众号的 HTML。

章节自动编号。

关键词自动加下划线。

代码块、表格、引用块都有现成样式。

也就是说，排版这件最耗心力的事，被一个 skill 固定下来了。

我不用每次纠结颜色、间距、字号。

只要把内容写好，剩下交给它。

这也是为什么这套流程能稳定产出。

**因为变量被压到了最少。**

## 每天的流程长什么样

准备都做好之后，每天就很轻了。

第一步，选个主题。

主题不靠硬想。

平时记下来的、最近踩的坑、刚学会的东西，都是素材。

第二步，把主题丢给 AI，让它生成文章。

因为项目里有一份写好的账号定位和风格说明，AI 写出来的东西不会跑偏。

第三步，调用 gzh-design 这个 skill，把文章排成 HTML。

第四步，浏览器打开生成的 HTML，点一下复制，粘到公众号编辑器。

整个过程，真正花时间的，其实就是第一步想主题，和最后扫一眼内容。

其他都是机械动作。

几分钟就够。

## 为什么我故意不自动化发布

这里我要专门强调一句。

整个流程里，发布这一步，我故意没有自动化。

技术上是可以做的。

调用公众号的接口，写个脚本，一键发出去，完全行得通。

但我没有这么干。

原因是，发布是最后一道关。

AI 写的内容，偶尔会出问题。

可能是事实错了。

可能是一段话读着别扭。

可能是排版在某个地方崩了。

如果全自动，这些问题就直接发出去了。

我自己都不看一眼，读者看到的就更没法保证。

所以我宁可在最后一步慢一点。

打开 HTML 看一遍。

复制粘贴的时候顺手再扫一眼。

这一步看着多余，其实是整套流程里最重要的人工卡点。

**AI 帮我把效率拉满，但最后拍板的还是我。**

## 最后

这套流程的核心，不是某一个工具多强。

而是把写公众号这件事，拆成了几个固定的小动作。

一次性把环境配好。

每天只做选题和最后的把关。

中间的写作和排版，交给 AI 和 skill。

它不是一个多复杂的系统。

就是几个小工具串起来，每个解决一个具体问题。

但串起来之后，写公众号这件事，从一件很重的事，变成了一件每天几分钟就能完成的小事。

如果你也想试试，按上面四步配一遍就行。

配好之后，剩下的就是每天花几分钟，写一篇。
