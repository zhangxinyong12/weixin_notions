# 我做了一个小工具：把 AI 写的 Markdown 一键变成公众号排版

这两天我踩了一个很小，但很烦的坑。

AI 帮我把文章写好了。

内容是 Markdown。

我复制到微信公众号编辑器里，发现样式全没了。

标题不像标题。

表格变成一堆文字。

重点句也不突出。

代码块更不用说，基本没法看。

然后我才意识到一件事：

**AI 能帮我们写文章，但它写出来的 Markdown，不能直接变成微信公众号里的漂亮排版。**

中间还差一步。

这一步就是：

把 Markdown 转成微信能识别的富文本。

## 为什么 Markdown 不能直接粘到公众号？

Markdown 本质上是写作格式。

比如：

```md
## 这是标题

**这是重点句**

| 场景 | 建议 |
|---|---|
| 新项目 | Playwright |
```

这些符号对程序员很友好。

但微信公众号编辑器不会自动帮你把它渲染成漂亮样式。

你直接粘进去，它就只是普通文字。

真正能保留样式的，是浏览器里的富文本，也就是带内联样式的 HTML。

所以正确流程不是：

```text
AI 生成 Markdown -> 直接粘公众号
```

而是：

```text
AI 生成 Markdown -> 转成微信富文本 -> 粘贴公众号
```

我之前就是少了中间这一步。

## 我试过的几个方向

最开始我想过用 VitePress。

毕竟它也是 Markdown 写文章，然后生成网页。

但想了一下，不合适。

VitePress 适合做博客站点，不适合做微信公众号排版。

公众号编辑器对外部 CSS、JS、class 都不友好。

你网页上看起来很漂亮，复制到公众号里不一定保得住。

然后我又看了一些现成工具，比如 doocs/md、mdnice。

这些工具很好用。

但我自己的需求更简单：

我只想有一个本地页面。

左边粘 Markdown。

右边看预览。

点一下复制。

再粘到公众号编辑器。

不登录。

不安装。

不依赖服务器。

最好以后我让 AI 写文章，也能按这个格式稳定输出。

于是就做了一个很小的本地工具。

## 这个工具怎么用？

工具路径是：

```text
tools/wechat-md-html/index.html
```

它就是一个普通 HTML 文件。

直接用浏览器打开就能用。

页面很简单：

左边是 Markdown 输入框。

右边是微信预览。

右侧有一个“复制”按钮。

点完之后，到微信公众号编辑器正文区直接粘贴。

> [!TIP]
> 注意，不是复制 Markdown 原文，而是复制右侧预览里的富文本。

## 我给它做了哪些样式？

我没有把它做成特别花的排版工具。

因为公众号文章最重要的是读起来舒服，不是每一屏都在炫技。

目前主要处理这些内容：

| Markdown 写法 | 转换后的效果 |
|---|---|
| `# 标题` | 文章主标题 |
| `## 二级标题` | 带强调色的章节标题 |
| `**重点句**` | 蓝色重点结论块 |
| `> [!NOTE]` | 说明提示块 |
| `> [!TIP]` | 建议提示块 |
| 表格 | 微信可粘贴的样式表格 |
| 代码块 | 深色代码块 |
| 行内代码 | 小号高亮代码样式 |

也就是说，以后我写文章时，不需要在 Markdown 里写颜色。

只要写清楚结构。

颜色、间距、边框、表格样式都交给转换器处理。

## 为什么不直接让 AI 写 HTML？

这个问题我也想过。

既然公众号要 HTML，那让 AI 直接输出 HTML 不就完了？

但我最后还是决定：

**原稿继续用 Markdown，发布时再转 HTML。**

原因很简单。

Markdown 好改。

Markdown 好读。

Markdown 适合长期保存。

同一篇文章以后还可以发到博客、知识库、GitHub。

如果一开始就是一大坨 HTML，后面改一句话都难受。

所以我现在把文章拆成两层：

| 层级 | 用途 |
|---|---|
| Markdown | 写作、修改、归档 |
| 富文本 HTML | 复制到公众号发布 |

这样更稳。

## 这个工具解决了什么小问题？

它解决的不是大问题。

但它解决了一个每天都会烦人的小问题。

以前流程是：

AI 写完文章。

我粘到公众号。

格式丢了。

手动调标题。

手动调表格。

手动加重点。

调完之后又觉得不好看。

现在流程变成：

```text
AI 写 Markdown
打开本地转换器
粘贴 Markdown
点复制
粘贴公众号
手机预览
```

这个变化不大。

但对经常写公众号的人来说，能省不少心智负担。

## AI 写文章时，也要配合这个工具

这个工具不是魔法。

它依赖 Markdown 本身写得清楚。

如果文章全是大段文字，没有标题、没有表格、没有重点句，转换出来也不会突然变高级。

所以以后我会让 AI 按这个规则写：

第一，章节用 `##`。

第二，关键结论用 `**加粗**`。

第三，适合对比的内容尽量用表格。

第四，提醒和说明用提示块。

比如：

```md
> [!NOTE]
> 这里放一条说明。

> [!TIP]
> 这里放一个建议。

> [!WARN]
> 这里放一个提醒。
```

这样 AI 写出来的 Markdown，不只是“文字内容”，还是有结构的内容。

结构清楚了，排版工具才有发挥空间。

## 最后

我现在越来越觉得，AI 时代很多效率提升，不是靠一个巨大的系统完成的。

反而是靠一堆很小的工具。

一个工具解决一个很具体的问题。

这次的问题就是：

AI 写好了 Markdown，但公众号不认 Markdown。

那就补一个转换器。

它不复杂。

但它让写作流程顺了很多。

对我这种正在用 AI 训练自己的人来说，这类小工具很有价值。

因为它不是单纯省几分钟。

它是在把一个重复动作固定下来。

以后每写一篇文章，都能少折腾一次。

**我的建议是：文章原稿继续用 Markdown，发布前用工具转成微信富文本。**

这比直接手动排版舒服，也比一开始就写 HTML 更适合长期维护。
