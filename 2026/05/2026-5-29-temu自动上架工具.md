# 我给 Temu 卖家做了两个自动化工具，上架效率翻了 10 倍

> 一个前端程序员的跨境电商接单实录：从发现痛点到交付产品的全过程。

---

上个月有个做 Temu 跨境电商的朋友找我诉苦。

他说每天要在店小秘上手动上架几百个产品——搜索商品、填标题、传图片、调尺寸、点发布，一个产品至少 5 分钟。每天上架 100 个就是 8 个小时，眼睛盯着屏幕，手指机械地点。

"你就不能帮我写个工具自动搞？"

我以前只写网页。Electron 桌面应用、浏览器插件、Playwright 自动化——这些我都没碰过。但我想试试。

两周后，我交给他两个工具。第一个帮他自动上架，第二个帮他批量管理流量加速器。现在他 100 个产品 40 分钟搞定。

---

## 第一个工具：店小秘自动上架

### 痛点有多痛

做 Temu 全托管的卖家，上架流程基本是这样的：

1. 在店小秘里搜索目标商品
2. 点编辑，填写产品标题（中文 + 英文）
3. 上传颜色图片
4. 把图片调整到平台要求的尺寸（1340×1785）
5. 刷新素材图，清理旧的 SKC 轮播图
6. 点发布

一个产品走完这套流程，快的 3 分钟，慢的 10 分钟。如果一天要上 200 个产品……

问题是：这些操作全是重复的。数据在 Excel 里，图片在文件夹里，每一步都是机械的点击和填写。人就变成了一个"人肉脚本"。

### 我做了什么

我用 Electron + Playwright 做了一个桌面应用。核心思路很简单：**让程序模拟人的操作，代替人去点**。

工作流程：

1. 打开 Excel，读取每行数据（搜索标题、产品标题、英文标题、文件夹名字）
2. 启动浏览器，自动导航到店小秘
3. 逐行执行：搜索商品 → 点编辑 → 填标题 → 上传图片 → 调整尺寸 → 清理轮播图 → 发布
4. 每发布完一个产品，自动清空相册释放空间
5. 全程显示进度条，哪个成功哪个失败一目了然

还加了一个图片水印功能——批量给产品图加上店铺 Logo，支持拖拽预览水印位置。

### 技术上踩的坑

说实话，这个项目对我挑战不小。我之前只做前端，没写过 Electron 主进程，没用过 Playwright 做浏览器自动化。

几个印象深的坑：

**页面元素定位不稳定。** 店小秘的页面结构经常变，class 名是动态生成的，今天能找到的元素明天可能就变了。最后我用的是组合定位策略——文字内容 + CSS 选择器 + 等待策略，三重保险。

**图片上传是个大坑。** Playwright 的文件上传 API 在某些场景下不生效，最后是通过拦截 file input 的事件，直接往里面塞文件路径解决的。

**反爬检测。** 操作太快会被检测，所以每步之间加了随机延迟（300-800ms），模拟真人的操作节奏。

这些坑，如果让我一个人从零啃，可能要一个月。但有了 Claude Code 帮忙，从搭项目框架到调试每个自动化步骤，效率高了很多。遇到 Playwright API 不熟的，直接问；遇到元素定位不上的，把页面截图丢给 AI 分析。两周搞定了。

## 第二个工具：流量加速器批量管理

### 又一个"重复点击地狱"

Temu 卖家中心有个功能叫"流量加速器"——相当于花钱买曝光。卖家可以选择降低申报价格来换取更多流量。

问题在于，每个产品都要单独开启：点"立即开启" → 选时长 → 选等级 → 点"加速" → 确认。一个产品 5 次点击。

如果你有 500 个产品要开启呢？2500 次点击。

更痛苦的是关闭。开启还好说，关闭要走另一条路：点进产品详情 → 切到"近期流量加速效果"tab → 点"停止流量加速" → 确认。每个产品至少 6 次点击，500 个就是 3000 次。

朋友说他有一次关了一下午，手指都酸了。

### Chrome 插件的解法

这次我选了 Chrome 插件方案。因为卖家中心是个网页，在网页里注入按钮，直接操作 DOM，比桌面应用更轻量。

用 Plasmo 框架（React 的 Chrome 插件开发框架），做了这么个东西：

- 在流量分析页面注入两个按钮："批量开启"和"批量关闭"
- 支持"当前页"或"所有页"两种范围
- 开启/关闭的每个步骤全自动：点按钮 → 选选项 → 确认 → 下一个
- 页面右上角显示实时进度：已处理、成功、失败
- 随时可以停

还有一个技术细节比较有意思：为了知道列表有多少页、每页多少条数据，我 Hook 了页面自身的 `fetch` 和 `XMLHttpRequest`，拦截 API 返回的分页信息。不用额外调接口，页面自己的数据就够用了。

## 从"写页面"到"做产品"

这两个项目对我来说不只是接了一单。

它让我真切体会到一件事：**程序员的技能边界，比你以为的宽得多。**

我之前觉得自己就是个写 React 的。Electron？没写过。Playwright 自动化？没碰过。Chrome 插件？没做过。但真正动手了才发现，前端的基础能力——DOM 操作、事件处理、异步编程、组件化思维——在这些场景里全都用得上。

区别只是：以前我给别人的产品写页面，现在我直接解决一个商业问题。

AI 在这里面帮了大忙。不是 AI 替我写代码，是 AI 帮我快速跨过了那些"我不知道我不知道"的门槛。Electron 主进程怎么搭？Playwright 怎么处理文件上传？Chrome 插件的 content script 和 background script 怎么通信？这些问题以前要翻半天文档，现在几分钟就能搞明白。

**但关键的决策还是得自己来。** 比如自动化步骤怎么设计才不会触发风控、图片上传用哪种策略最稳定、UI 怎么设计卖家才用得明白——这些是理解业务之后才能做的判断。

## 写给同样在转型的程序员

很多人问我：前端程序员能做什么副业？

我的答案越来越清晰：**找到那些"重复劳动密集 + 技术门槛不高但一般人搞不定"的场景，用代码干掉重复。**

跨境电商就是一个很好的领域。卖家懂运营懂选品，但他们不懂数据处理，不懂自动化，不懂批量操作。你不需要成为跨境电商专家，你只需要看懂他们的工作流程，找到那个可以自动化的环节。

这比做一个没人用的 Side Project 强多了。
