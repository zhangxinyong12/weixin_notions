# daydayago 公众号内容仓库

这是 `daydayago` 微信公众号的文章创作与排版仓库。

账号定位：

> 35 岁前端程序员，正在用 AI 把自己从“写页面的人”，重新训练成“能做产品、能接单、能独立开发的人”。

这里主要存放公众号 Markdown 原稿，以及一个本地 Markdown 转微信公众号排版的小工具。

## 内容方向

文章主要围绕这些主题：

- AI 工具使用
- AI 编程与自动化
- 前端程序员转型
- 独立开发
- 接单变现
- 产品思考
- 实战工具复盘

核心问题是：

**中年程序员怎么借助 AI 重新掌握主动权。**

## 微信公众号排版工具

仓库内置了一个本地排版工具：

```text
tools/wechat-md-html/index.html
```

它的作用是：

```text
Markdown 原稿 -> 微信公众号可粘贴的富文本
```

使用方式：

1. 用浏览器打开 `tools/wechat-md-html/index.html`
2. 左侧粘贴 Markdown
3. 右侧查看预览
4. 点击右侧“复制”
5. 粘贴到微信公众号编辑器正文区

这个工具不需要安装依赖，不需要启动服务，也不需要登录。

## Markdown 写作约定

为了让文章在转换后更好看，推荐使用这些写法：

```md
# 文章标题

开头正文。

**一句重要结论。**

## 二级标题

正文内容。

> [!NOTE]
> 这是说明块。

> [!TIP]
> 这是建议块。

> [!WARN]
> 这是提醒块。

| 场景 | 建议 |
|---|---|
| 新项目 | Playwright |
| 小脚本 | Puppeteer |
```

原则：

- 原稿继续使用 Markdown
- 不在 Markdown 里硬写 HTML 样式
- 样式交给转换工具统一处理
- 公众号发布前复制右侧富文本，而不是复制 Markdown 原文

## 当前工作流

```text
1. 用 AI 或手动写 Markdown 原稿
2. 保存到 2026/月份/ 目录
3. 打开 tools/wechat-md-html/index.html
4. 粘贴 Markdown 并预览
5. 复制富文本到微信公众号编辑器
6. 手机预览后发布
```

## 说明

这个仓库不是一个通用 CMS，也不是完整博客系统。

它更像一个个人公众号内容工作台：

- Markdown 用来写作和归档
- 本地工具用来解决公众号排版
- AI 协作规则用来保持内容方向一致

后续如果需要网站归档，可以再基于 Markdown 增加 VitePress、Astro 或其他静态站点生成器。
